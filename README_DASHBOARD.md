🏮 营造 · 天枢 (Ying-Zao Tian-Khu)
—— 古建筑大木作数字化保护与交互解构平台
项目愿景：传承数字化，赋予古建筑第二次生命。本系统旨在通过 GIS 测绘、3D 榫卯解构及区块链式存证技术，将碎片化的古建筑营造知识转化为可交互、可视化、可回响的数字档案。

🛠️ 核心模块功能与优势说明 (Module Documentation)
1. 寻古舆图 (Ancient GIS Tracking)
功能描述：基于高德 API 的数字化宣纸长卷。集成全国 12+ 处顶级木构节点（如唐代佛光寺、辽代应县木塔）。

技术特性：采用十字准星测绘标点替代传统圆点，点击可实现从全国俯瞰到省份区域的飞行聚焦（Fly-To）。

文档优势：

时空穿透性：实现了从宏观地理分布到微观建筑档案的无缝衔接。

高对比视觉：黑白红碑刻式 UI 设计，在大型显示器上具备极高的可读性与权威感。

2. 营造解构 · 斗拱 (3D Structural Lab)
功能描述：基于 Three.js 的交互式零件实验室。对辽代观音阁斗拱进行 0.1mm 级的数字化孪生。

技术特性：支持滚轮缩放查看细部、零件手动拖拽拆解及一键重装复位。

文档优势：

逻辑传递：通过手动拆解，让抽象的“材份制”与“榫卯结构”变得触手可及。

数字档案感：实时显示 Zoom 缩放倍率，模拟实验室精密检测流程。

3. 肆时营造 (Chronicle Camp) —— 【右一】
功能描述：动态展示大木作施工全过程。涵盖：伐木制材、弹线划正、榫卯凿作、升架合龙四大工序。

交互逻辑：采用动态步进系统，模拟营造工程的流转，点亮当前正在进行的工序。

文档优势：

工序可视化：将静态的建筑转变为动态的建设流程，体现了中国古建筑“以材为祖”的时间属性。

4. 时空回响 (Echo Resonance) —— 【右二】
功能描述：实时互动监测模块。统计当前线上正在查看或关注特定建筑的“回响值”。

交互逻辑：模拟宣纸墨滴扩散动效，数值随实时访问频率跳动。

文档优势：

墨韵共赏：将枯燥的用户统计转化为极具意境的“墨痕回响”，体现了文化遗产在数字时空的生命力。

5. 数字朱印 (Digital Seal) —— 【右三】
功能描述：金石印谱式数字存证系统。为每一份数字档案生成唯一的数字化验证 Hash。

交互逻辑：点击触发**“朱砂盖章”**反馈，生成不可篡改的认证编码。

文档优势：

文化契合度：将现代区块链存证理念与传统印章文化结合，强化了数字档案的严肃性。

6. 营造漫谈 (Scroll Log) —— 【左一】
功能描述：系统实时抓取的文献流与研学动态，以打字机流式输出。

文档优势：持续的信息流动确保了界面的实时性（Live Data），模拟学术研究的动态进展。

7. 四时意象 (Seasonal View) —— 【左二】
功能描述：环境渲染模拟。根据节气变化自动切换古建筑在阴晴雨雪下的视觉参数。

文档优势：环境共生论，展示了木构建筑随四季更迭而产生的物理与美学变化。

8. 寻味中国 (Palette Box) —— 【左三】
功能描述：传统色彩提取模块。数字化分析古建彩画中的矿物颜料成分（如朱砂、石青）。

文档优势：色彩考古学，为古建筑修复提供科学的数字化色度标准。

🎨 系统设计语言 (Design Identity)
红墙白宣 (Imperial Palette)：核心色值采用 #9B2E2E (宫墙红) 搭配 #FCFAF5 (宣纸黄)，营造宫廷档案的质感。

金石排版 (Epigraphic Layout)：大量使用竖排文字（Writing-mode: vertical）与加粗黑体，模拟碑刻感。

数字化测绘 (Scanning Aesthetic)：在宣纸底纹上穿插 3D 扫描线、坐标轴与 Hash 码，体现“科技+古建”的碰撞。

🚀 技术架构 (Tech Architecture)
View 层：Vue 3 + Tailwind CSS (用于极速响应式布局)

3D 层：Three.js + GSAP (用于平滑的构件复位动画)

GIS 层：AMap JSAPI 2.0 (通过 CSS 滤镜定制宣纸地图效果)

数据层：architectureData.js (实现数据与展示逻辑的深度解耦)
